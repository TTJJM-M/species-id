# 0002: macOS 桌面托管 + cloudflared 隧道的轻后端

后端最初定在阿里云同区部署（函数计算或轻量容器，与百炼内网互通）。但确认为自用不发布后，云端部署的常驻成本（月租、备案、运维）与零请求时段的浪费不成比例。

改为在本机 Mac 上用 FastAPI + cloudflared free tunnel 暴露公网 HTTPS 端点，手机 app 通过 cloudflared 随机域名访问。每次使用前手动启动 `start.sh` 拉起服务，用完关闭。

代价：Mac 需开机且不睡眠时可用、每次手动填写 cloudflared 随机域名到 app 设置。换取的收益：零服务器成本、零冷启动延迟、调试即本地。自用规模完全可接受。

考虑过的选项：阿里云函数计算（冷启动影响体验）、阿里云轻量服务器（月租浪费）、Mac 本地仅局域网（出门不可用）。
